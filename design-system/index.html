Capability-aware routing

Every ticket carries a tier. Every agent declares what it can take.

A local 30B model is a reliable teammate for fully-decomposed work and unreliable on open-ended work. Tasca encodes that boundary as a first-class tier system, then assigns work an agent can actually complete — prompted the way that agent needs.

No reasoning, just coding. Exact files, IO contract, every edge case enumerated.

Per-tier prompt scaffolding

Low and basic templates constrain the tool surface, cap turns, and forbid open-ended planning: implement exactly the spec, don't redesign.

Atomic claim, no double-dispatch

A conditional update claims an agent only while it has free capacity. Two dispatchers race; one wins; the loser re-queries. The single GPU never over-commits.

Escalation, never silent failure

A failed attempt flips the ticket to needs_attention with the reason and a one-click escalate to a higher tier or a cloud agent. No ticket is ever dropped.

Review-driven automation

Reviews drive the board. Automatically.

An agent opens a PR on the ticket's branch; it links back automatically. From there, GitHub review events move the ticket — event-sourced on PR state, idempotent, and reconciled if a webhook is ever missed.

Trust & isolation

External input never reaches a shell unguarded.

Coding agents run real commands. Tasca treats untrusted, externally-filed work as a first-class threat and gates it at the team layer — before any dispatch reaches an executor.

Propose-only guests

External clients can file and comment, never execute. An internal human promotes a ticket to agent-ready before any run. Untrusted input can't trigger work on its own.

Sandboxed runs

Any external-touch execution runs in an ephemeral container — no host filesystem, no secrets, egress-restricted. The GPU stays on the host; the sandbox reaches it over the network.

Defaults flipped to supervised

Permissive agent defaults are switched to approval-gated on any non-trusted path. Self-hosted means your keys, your infra, your audit trail — every actor logged uniformly.
